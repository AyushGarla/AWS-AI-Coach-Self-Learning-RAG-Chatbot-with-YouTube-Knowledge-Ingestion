## Amazon Web Services (AWS) — Overview
Amazon Web Services (AWS) is a comprehensive cloud computing platform offering over 200 fully managed services including compute, storage, networking, analytics, machine learning, databases, security, DevOps, and serverless technologies. AWS operates globally through Regions and Availability Zones, ensuring high availability, low latency, and fault tolerance.

### Key Benefits
- Global Infrastructure  
- Pay-as-you-go pricing  
- On-demand scalability  
- High durability & availability  
- Strong security and compliance  
- Fully managed services  

---

## Amazon S3 (Simple Storage Service)
Amazon S3 is a massively scalable, highly durable object storage service. It is used for backups, logs, static website hosting, media storage, machine learning datasets, and more.

### Concepts
- **Bucket**: Container for objects  
- **Object**: Data + metadata + key  
- **Object Size Limit**: Up to 5 TB  
- **Namespace**: Bucket names are globally unique  

### Storage Classes
- S3 Standard  
- S3 Intelligent-Tiering  
- S3 Standard-IA  
- S3 One Zone-IA  
- Glacier Instant Retrieval  
- Glacier Flexible Retrieval  
- Glacier Deep Archive  

### Features
- Versioning  
- Lifecycle Rules  
- Cross-Region Replication  
- S3 Event Notifications (SQS, SNS, Lambda)  
- Pre-Signed URLs  
- Server-Side Encryption (SSE-S3, SSE-KMS)  

### Common Use Cases
- Data lake storage  
- Backup & archive  
- Media hosting  
- ML training data  
- Website file hosting  

---

## Amazon EC2 (Elastic Compute Cloud)
EC2 provides resizable virtual machines (instances) in the cloud.

### Instance Families
- **General Purpose** — T, M  
- **Compute Optimized** — C  
- **Memory Optimized** — R, X, Z  
- **GPU Instances** — P, G  
- **Storage Optimized** — I, D  

### Features
- Security Groups  
- EBS Volumes (gp3, io2, st1, sc1)  
- Elastic IPs  
- Auto Scaling Groups  
- Placement Groups (Cluster, Spread, Partition)  

### Pricing Models
- On-Demand  
- Reserved Instances  
- Spot Instances  
- Savings Plans  

---

## Amazon VPC (Virtual Private Cloud)
VPC allows you to create isolated virtual networks inside AWS.

### Core Components
- Subnets (public, private)  
- Route Tables  
- Internet Gateway  
- NAT Gateway  
- Security Groups  
- Network ACLs  
- VPC Peering  
- Transit Gateway  
- VPC Endpoints (Gateway & Interface)  

### Features
- Completely isolated network  
- IPv4/IPv6 support  
- Flow Logs  
- Custom routing  
- Network segmentation  

---

## AWS Lambda
Lambda is a serverless compute service that runs code without provisioning servers.

### Characteristics
- Fully managed  
- Pay per request + execution time  
- Supports many runtimes (Python, Node.js, Java, Go, etc.)  
- Scales automatically  

### Event Sources
- API Gateway  
- S3  
- DynamoDB Streams  
- EventBridge  
- SQS/SNS  

### Use Cases
- ETL and data processing  
- Real-time file processing  
- REST APIs  
- Chatbots and automation  

---

## Amazon DynamoDB
DynamoDB is a fully managed NoSQL database offering single-digit millisecond performance at any scale.

### Concepts
- Table → Items → Attributes  
- Partition Key, Sort Key  
- GSI / LSI  
- Streams  
- On-demand & Provisioned modes  

### Features
- Auto Scaling  
- DynamoDB Streams  
- Time-To-Live (TTL)  
- Global Tables  
- Fine-grained access control  

---

## Amazon RDS (Relational Database Service)
RDS is a managed relational database service supporting multiple engines.

### Supported Engines
- MySQL  
- PostgreSQL  
- MariaDB  
- Oracle  
- SQL Server  
- Aurora (MySQL/Postgres compatible)  

### Features
- Automated backups  
- Multi-AZ deployments  
- Read replicas  
- Automated patching  
- Monitoring & Performance Insights

## Amazon CloudFront
A global Content Delivery Network (CDN) that caches content at edge locations.

### Features
- Low latency content delivery  
- Origin Shield  
- Signed URLs & Cookies  
- WAF Integration  
- Supports S3, EC2, ALB, API Gateway as origins  

### Use Cases
- Media streaming  
- Static website delivery  
- API acceleration
